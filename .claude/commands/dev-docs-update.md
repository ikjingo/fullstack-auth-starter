컨텍스트 한계에 접근하고 있습니다. 컨텍스트 리셋 후 원활한 작업 재개를 위해 개발 문서를 업데이트해주세요.

## 필수 업데이트

### 1. 활성 작업 문서 업데이트

`/dev/active/`의 각 작업에 대해:

**[task-name]-context.md 업데이트:**
- 현재 구현 상태
- 이번 세션에서 내린 주요 결정
- 수정된 파일들과 이유
- 발견된 차단 요소나 이슈
- 다음 즉시 단계
- Last Updated 타임스탬프

**[task-name]-tasks.md 업데이트:**
- 완료된 작업은 ✅로 표시
- 새로 발견된 작업 추가
- 진행 중인 작업 상태 업데이트
- 필요시 우선순위 재정렬

### 2. 세션 컨텍스트 캡처

다음과 관련된 정보를 포함하세요:
- 해결한 복잡한 문제들
- 내린 아키텍처 결정
- 찾아서 수정한 까다로운 버그
- 발견한 통합 포인트
- 사용한 테스트 접근법
- 적용한 성능 최적화

### 3. 메모리 업데이트 (해당되는 경우)

- 프로젝트 메모리/문서에 새로운 패턴이나 솔루션 저장
- 발견한 엔티티 관계 업데이트
- 시스템 동작에 대한 관찰 추가

### 4. 미완료 작업 문서화

- 컨텍스트 한계에 접근했을 때 작업 중이던 것
- 부분적으로 완료된 기능의 정확한 상태
- 재시작 시 실행해야 할 명령어
- 영구적 수정이 필요한 임시 해결책

### 5. 핸드오프 노트 생성

새 대화로 전환하는 경우:
- 편집 중이던 정확한 파일과 라인
- 현재 변경의 목표
- 주의가 필요한 커밋되지 않은 변경사항
- 작업을 검증할 테스트 명령어

## 추가 컨텍스트: $ARGUMENTS

**우선순위**: 코드만으로는 재발견하거나 재구성하기 어려운 정보를 캡처하는 데 집중하세요.

## 완료 후 요약

업데이트된 내용을 간략히 요약해주세요:
- 완료된 작업 수
- 남은 작업 수
- 주요 결정사항
- 다음 세션 시작 시 해야 할 것
