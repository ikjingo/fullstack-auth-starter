# 빌드 및 에러 수정

대상: $ARGUMENTS

## 지시사항

프로젝트 빌드를 실행하고 발견된 모든 에러를 수정합니다.

### 1. 빌드 대상 결정

`$ARGUMENTS` 값에 따라:
- `backend` 또는 비어있으면서 백엔드 변경이 있으면: 백엔드 빌드
- `frontend` 또는 비어있으면서 프론트엔드 변경이 있으면: 프론트엔드 빌드
- `all` 또는 비어있으면: 둘 다 빌드

### 2. 빌드 실행

#### Backend (Kotlin)
```bash
cd backend && ./gradlew compileKotlin
```

#### Frontend (TypeScript)
```bash
cd frontend && npx tsc --noEmit
```

### 3. 에러 수정 프로세스

1. **에러 분류**
   - 타입 에러
   - 임포트 에러
   - 문법 에러
   - 런타임 에러

2. **우선순위 결정**
   - 의존성 순서대로 수정 (다른 파일에서 참조하는 파일 먼저)
   - 간단한 에러부터 복잡한 에러 순

3. **수정 진행**
   - 각 에러를 하나씩 수정
   - 수정 후 재빌드로 검증
   - 새로운 에러 발생 시 함께 처리

4. **반복**
   - 모든 에러가 해결될 때까지 반복
   - 최대 5회 반복 후에도 해결 안 되면 사용자에게 보고

### 4. 결과 보고

```markdown
## 빌드 결과

### 초기 상태
- 백엔드 에러: X개
- 프론트엔드 에러: X개

### 수정 내역
1. `path/to/file.ts:line` - 에러 설명 -> 수정 내용
2. ...

### 최종 상태
- 백엔드: 빌드 성공/실패
- 프론트엔드: 빌드 성공/실패

### 남은 이슈 (있는 경우)
- 수동 확인 필요한 항목
```

## 주의사항
- 에러 수정 시 기존 로직을 변경하지 않도록 주의
- 타입 에러 수정 시 `any` 타입 사용 지양
- 임포트 경로 수정 시 프로젝트 컨벤션 준수
